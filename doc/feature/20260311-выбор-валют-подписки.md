# Кастомный выбор валют для подписки

## Цель

Дать пользователям Telegram-бота возможность самостоятельно выбирать, по каким валютам получать курсы в подписке и ручных запросах. Сейчас все получают одинаковый набор из 5 валют (USD, EUR, GBP, JPY, CNY). После реализации каждый пользователь сможет настроить свой набор из 10 популярных валют.

## Текущее поведение

### Жёстко зашитый набор валют

Файл `src/bot/ExchangeRatesBot.App/Phrases/BotPhrases.cs`:
```csharp
public static string[] Valutes { get; } = new string[] { "USD", "EUR", "GBP", "JPY", "CNY" };
```

Этот массив используется в двух местах:
1. **CommandService** (строки 116, 123) -- при обработке команд `/valutesevendays` и `/valuteoneday`
2. **JobsSendMessageUsers** (строка 66) -- при рассылке подписчикам

### Модель пользователя не хранит предпочтения

`UserDb` содержит только флаг `Subscribe` (bool) -- нет возможности хранить выбранные валюты.

### Рассылка одинакова для всех

`JobsSendMessageUsers.TaskListUsers()` формирует одно сообщение `messageValute.GetValuteMessage(1, BotPhrases.Valutes, cancel)` и рассылает его всем подписчикам. Нет персонализации.

## Желаемое поведение

### Новая команда `/currencies`

Пользователь отправляет `/currencies` и получает inline-клавиатуру с 10 популярными валютами. Каждая кнопка отражает текущий статус выбора.

### Пример сообщения с клавиатурой

```
Выберите валюты для отслеживания:

[v USD] [v EUR] [  GBP]
[  JPY] [v CNY] [  CHF]
[  CAD] [  AUD] [  TRY]
[  BYN]
[   Сохранить   ]
```

Где `v` означает выбранную валюту. В реальном Telegram это будет выглядеть так:
- Выбранная: `[check] USD`
- Не выбранная: `[empty] USD`

### Переключение по нажатию

При нажатии на кнопку валюты -- её статус меняется на противоположный. Клавиатура обновляется (EditMessageReplyMarkupAsync) без отправки нового сообщения.

### Сохранение

При нажатии "Сохранить" -- выбранные валюты записываются в БД. Бот отвечает подтверждением.

### Персонализированная рассылка и запросы

Команды `/valuteoneday`, `/valutesevendays` и фоновая рассылка используют выбранные пользователем валюты вместо `BotPhrases.Valutes`.

## Миграция БД

### Новое поле в UserDb

Добавить строковое поле `Currencies` в таблицу `Users`. Формат хранения -- коды валют через запятую: `"USD,EUR,GBP"`.

Обоснование выбора строки вместо отдельной таблицы:
- Количество возможных валют фиксировано (34 из API ЦБ РФ)
- Набор пользователя -- это простой список кодов, нет дополнительных атрибутов
- Нет необходимости делать SQL-запросы по отдельным валютам
- Простота реализации и миграции
- Для SQLite с текущим масштабом (десятки/сотни пользователей) это оптимальное решение

### Значение по умолчанию

`Currencies` должен быть `null` (nullable string). `null` означает: "пользователь не настраивал валюты, использовать дефолтный набор".

Дефолтный набор: `"USD,EUR,GBP,JPY,CNY"` (текущие 5 валют из `BotPhrases.Valutes`).

Для хранения промежуточного состояния выбора (между нажатиями кнопок и сохранением) использовать статический `ConcurrentDictionary`:

```csharp

// В начале класса:
private static readonly ConcurrentDictionary<long, HashSet<string>> _pendingSelections = new();
```

Обоснование: состояние нужно только на время сессии выбора (от /currencies до "Сохранить"). Если бот перезапустится -- пользователь просто вызовет /currencies заново. ConcurrentDictionary безопасен для многопоточного доступа.

#### 8.3 Обновить метод CallbackMessageCommand

Добавить обработку callback для toggle и save. Важно: для callback-запросов нужно установить CurrentUser так же, как для обычных сообщений.

```csharp
private async Task CallbackMessageCommand(Update update)
{
    var callbackData = update.CallbackQuery.Data;

    // Установить текущего пользователя для callback
    var chatId = update.CallbackQuery.From.Id;
    await _userControl.SetUser(chatId);

    // Toggle валюты
    if (callbackData.StartsWith("toggle_"))
    {
        var currencyCode = callbackData.Substring(7); // "toggle_USD" -> "USD"
        await HandleToggleCurrency(update, chatId, currencyCode);
        return;
    }

    switch (callbackData)
    {
        case "save_currencies":
            await HandleSaveCurrencies(update, chatId);
            break;

        case "Подписаться":
            await _userControl.SubscribeUpdate(_userControl.CurrentUser.ChatId, true, CancellationToken.None);
            await _updateService.EchoTextMessageAsync(update, BotPhrases.SubscribeTrue, default);
            break;

        case "Отписаться":
            await _userControl.SubscribeUpdate(_userControl.CurrentUser.ChatId, false, CancellationToken.None);
            await _updateService.EchoTextMessageAsync(update, BotPhrases.SubscribeFalse, default);
            break;
    }
}
```

#### 8.4 Новые приватные методы

```csharp
private async Task HandleToggleCurrency(Update update, long chatId, string currencyCode)
{
    if (!_pendingSelections.ContainsKey(chatId))
    {
        // Сессия выбора истекла (перезапуск бота) -- инициализировать из БД
        var currentCurrencies = _userControl.GetUserCurrencies(chatId);
        _pendingSelections[chatId] = new HashSet<string>(currentCurrencies);
    }

    var selection = _pendingSelections[chatId];
    if (selection.Contains(currencyCode))
        selection.Remove(currencyCode);
    else
        selection.Add(currencyCode);

    // Обновить клавиатуру без отправки нового сообщения
    var botService = /* получить из DI -- см. примечание ниже */;
    await botService.Client.EditMessageReplyMarkupAsync(
        chatId: update.CallbackQuery.Message.Chat.Id,
        messageId: update.CallbackQuery.Message.MessageId,
        replyMarkup: new InlineKeyboardMarkup(CurrenciesKeyboard(chatId)));

    // Ответить на callback, чтобы убрать "часики" на кнопке
    await botService.Client.AnswerCallbackQueryAsync(update.CallbackQuery.Id);
}

private async Task HandleSaveCurrencies(Update update, long chatId)
{
    if (!_pendingSelections.ContainsKey(chatId) || _pendingSelections[chatId].Count == 0)
    {
        await _updateService.EchoTextMessageAsync(update, BotPhrases.CurrenciesEmpty, default);
        return;
    }

    var selected = _pendingSelections[chatId];
    var currenciesString = string.Join(",", selected);
    await _userControl.UpdateCurrencies(chatId, currenciesString, CancellationToken.None);

    // Очистить временное состояние
    _pendingSelections.TryRemove(chatId, out _);

    await _updateService.EchoTextMessageAsync(
        update,
        BotPhrases.CurrenciesSaved + currenciesString,
        default);
}

private List<List<InlineKeyboardButton>> CurrenciesKeyboard(long chatId)
{
    var selected = _pendingSelections.ContainsKey(chatId)
        ? _pendingSelections[chatId]
        : new HashSet<string>();

    var rows = new List<List<InlineKeyboardButton>>();
    var currentRow = new List<InlineKeyboardButton>();

    foreach (var currency in BotPhrases.AvailableCurrencies)
    {
        var isSelected = selected.Contains(currency);
        var label = isSelected ? $"\u2705 {currency}" : $"\u2610 {currency}";
        currentRow.Add(InlineKeyboardButton.WithCallbackData(label, $"toggle_{currency}"));

        if (currentRow.Count == 3)  // по 3 кнопки в ряд
        {
            rows.Add(currentRow);
            currentRow = new List<InlineKeyboardButton>();
        }
    }
    if (currentRow.Count > 0)
        rows.Add(currentRow);

    // Кнопка "Сохранить" -- отдельный ряд
    rows.Add(new List<InlineKeyboardButton>
    {
        InlineKeyboardButton.WithCallbackData("\u2705 Сохранить", "save_currencies")
    });

    return rows;
}
```

### 9. ExchangeRatesBot.Maintenance/Jobs/JobsSendMessageUsers.cs

**Действие**: Персонализировать рассылку -- формировать сообщение для каждого пользователя индивидуально.

Текущий код формирует одно сообщение для всех:
```csharp
var message = await messageValute.GetValuteMessage(1, BotPhrases.Valutes, cancel);
```

Изменить метод `TaskListUsers`:

```csharp
private async Task TaskListUsers(IServiceProvider scope, CancellationToken cancel)
{
    var botService = scope.GetRequiredService<IBotService>();
    var messageValute = scope.GetRequiredService<IMessageValute>();
    var repo = scope.GetRequiredService<IBaseRepositoryDb<UserDb>>();
    var usersCollectionDb = await repo.GetCollection(cancel);

    var users = usersCollectionDb.Where(u => u.Subscribe == true).ToArray();

    if (!users.Any()) return;

    // Группируем пользователей по набору валют для минимизации запросов к API
    var groups = users.GroupBy(u => u.Currencies ?? BotPhrases.DefaultCurrencies);

    foreach (var group in groups)
    {
        var currencies = group.Key.Split(',');
        var message = await messageValute.GetValuteMessage(1, currencies, cancel);

        foreach (var userDb in group)
        {
            await botService.Client.SendTextMessageAsync(
                userDb.ChatId, message, parseMode: ParseMode.Markdown);
        }
    }
}
```

Важная оптимизация: группировка по набору валют. Если 50 пользователей используют дефолтный набор, запрос к API выполнится один раз, а не 50.

## Обратная совместимость

1. **Миграция БД**: Столбец `Currencies` добавляется как nullable. Существующие записи остаются с `NULL`.
2. **NULL = дефолт**: Везде, где читается `Currencies`, проверяется на `null`. Если `null` -- используется `BotPhrases.Valutes` (текущий набор из 5 валют).
3. **Пользователи без настройки**: Ничего не меняется. Получают те же 5 валют, что и раньше.
4. **Старые callback-данные**: Существующие callback "Подписаться"/"Отписаться" продолжают работать.
5. **Авто-миграция**: `dataDb.Database.Migrate()` в Startup.Configure() применит новую миграцию автоматически.

## Риски и ограничения

### ConcurrentDictionary для временного состояния

`_pendingSelections` хранится в статическом поле. При перезапуске бота данные теряются. Это приемлемо: пользователь просто вызовет `/currencies` заново. Альтернатива (хранение в БД) излишне усложняет решение.

### Порядок валют при сохранении

Валюты в строке `Currencies` могут быть в произвольном порядке (зависит от `HashSet`). Это не влияет на функциональность, но сообщение "Выбранные валюты: ..." может показывать их не по алфавиту. При желании можно добавить сортировку перед `string.Join`.

### Ограничение Telegram на размер сообщения

При выборе всех 10 валют сообщение с 7-дневной историей может быть длинным. Telegram поддерживает до 4096 символов. Для 10 валют * 7 дней это около 2000-3000 символов -- укладывается.

### Thread safety

`CommandService` регистрируется как Scoped. Однако `_pendingSelections` -- статический `ConcurrentDictionary`, что обеспечивает потокобезопасность. Каждый chatId работает со своим `HashSet`, параллельный доступ к одному chatId маловероятен (один пользователь не может отправить два callback одновременно).

## Критерии приёмки

### Функциональные

- [ ] Команда `/currencies` отображает inline-клавиатуру с 10 валютами
- [ ] Текущий выбор пользователя (или дефолтный набор) отмечен галочками при открытии
- [ ] Нажатие на кнопку валюты переключает её состояние (toggle)
- [ ] Клавиатура обновляется без отправки нового сообщения (EditMessageReplyMarkupAsync)
- [ ] Кнопка "Сохранить" записывает выбор в БД
- [ ] Попытка сохранить пустой набор показывает предупреждение
- [ ] Команда `/valuteoneday` использует пользовательский набор валют
- [ ] Команда `/valutesevendays` использует пользовательский набор валют
- [ ] Рассылка использует пользовательский набор валют для каждого подписчика
- [ ] Новые пользователи (Currencies == null) получают дефолтный набор из 5 валют
- [ ] Существующие пользователи продолжают работать без изменений до первой настройки

### Нефункциональные

- [ ] Миграция применяется автоматически при запуске (авто-миграция в Startup.Configure)
- [ ] Рассылка группирует пользователей по набору валют (оптимизация запросов к API)
- [ ] Команда /start показывает /currencies в списке доступных команд
- [ ] Совместимость с Telegram.Bot v16.0.2 (не использовать API из v17+)
- [ ] Polling и Webhook режимы корректно обрабатывают callback-запросы
