# Расширенная статистика за период

## Цель

Добавить блок расчетной статистики (максимум, минимум, изменение за период) к команде `/valutesevendays` и полностью переработать формат рассылки подписчикам в `JobsSendMessageUsers`, заменив перечисление дневных курсов на компактную сводку с недельной динамикой.

## Текущее поведение

### Команда `/valutesevendays`

Выводит список курсов за 7 дней с разницей между соседними днями:

```
*Доллар США*
 USD/RUB

 03.03.2026 ---> 85.45
 04.03.2026 ---> 85.90  (+ *0.45)*
 05.03.2026 ---> 86.20  (+ *0.30)*
 ...
```

Формат: только хронологический список, без агрегированных показателей.

### Рассылка подписчикам (JobsSendMessageUsers)

Вызывает `messageValute.GetValuteMessage(1, BotPhrases.Valutes, cancel)` с `day=1`, получая курс только за текущий день для 5 валют (USD, EUR, GBP, JPY, CNY). Формат аналогичен однодневному выводу -- просто текущее значение без контекста динамики.

## Желаемое поведение

### Команда `/valutesevendays` -- добавить блок статистики

После существующего списка дневных курсов добавить блок:

```

 USD/RUB

 04.03.2026 ---> 85.90  (+ *0.45)*
 05.03.2026 ---> 86.20  (+ *0.30)*

 Макс: 86.20 (05 мар)

```

### Рассылка подписчикам -- новый формат сводки

Заменить текущий вывод на компактную сводку:

```
*Курсы ЦБ РФ* 10 марта

USD  85.45  -1.20 за неделю
EUR  92.30  +0.85 за неделю
GBP 108.12  без изм.
JPY   0.57  -0.02 за неделю
CNY  11.82  +0.15 за неделю
```

Каждая строка содержит: код валюты, текущий курс, изменение за 7 дней (или "без изм." при нулевом изменении).

#### 3.3. Новый метод `GetValuteSummaryMessage`

Реализует интерфейс `IMessageValute.GetValuteSummaryMessage`. Для каждой валюты из массива делает запрос с `day=8` (чтобы гарантированно получить 7 рабочих дней с учетом выходных), рассчитывает статистику и формирует строку сводки.

```csharp
public async Task<string> GetValuteSummaryMessage(
    string[] charCodesCollection, CancellationToken cancel)
{
    var dateHeader = DateTime.Now.ToString("d MMMM",
        System.Globalization.CultureInfo.GetCultureInfo("ru-RU"));
    var result = $"*Курсы ЦБ РФ* {dateHeader}\n\r\n\r";

    foreach (var charCode in charCodesCollection)
    {
        var valutesRoot = await _processingService.RequestProcessing(8, charCode, cancel);
        if (valutesRoot == null) continue;

        var v = valutesRoot.GetValuteModels
            .Select(item => new Valute
            {
                CharCode = item.CharCode,
                DateValute = item.DateValute,
                Name = item.Name,
                Value = item.Value
            })
            .GroupBy(e => e.DateValute)
            .Select(g => g.First())
            .OrderBy(e => e.DateValute)
            .ToList();

        if (v.Count < 2)
        {
            result += $"{charCode}  {v.FirstOrDefault()?.Value:0.00}  нет данных\n\r";
            continue;
        }

        var stats = CalculateStatistics(v);
        string changeStr;
        if (Math.Abs(stats.AbsoluteChange) < 0.005)
        {
            changeStr = "без изм.";
        }
        else
        {
            var sign = stats.AbsoluteChange >= 0 ? "+" : "";
            changeStr = $"{sign}{stats.AbsoluteChange:0.00} за неделю";
        }

        result += $"{stats.CharCode}  {stats.CurrentValue:0.00}  {changeStr}\n\r";
    }

    return result;
}
```

## Алгоритмы расчета

### Входные данные

Метод `CalculateStatistics` принимает `List<Valute>`, который уже:
- Дедуплицирован по `DateValute` (`.GroupBy(e => e.DateValute).Select(g => g.First())` -- уже реализовано в текущем коде)
- Отсортирован по `DateValute` по возрастанию (самая старая дата -- первый элемент)

### Максимум и минимум за период

```csharp
// Находим элемент с максимальным значением
var maxItem = valutes.OrderByDescending(v => v.Value).First();
stats.MaxValue = maxItem.Value;
stats.MaxDate = maxItem.DateValute;

// Находим элемент с минимальным значением
var minItem = valutes.OrderBy(v => v.Value).First();
stats.MinValue = minItem.Value;
stats.MinDate = minItem.DateValute;
```

Используется `OrderBy/OrderByDescending` + `First()` вместо `MaxBy`/`MinBy`, поскольку `MaxBy` доступен только начиная с .NET 6, а проект использует .NET 5.0.

### Изменение за период (абсолютное)

```csharp
var firstValue = valutes.First().Value;  // самый старый курс
var lastValue = valutes.Last().Value;    // самый свежий курс
stats.AbsoluteChange = lastValue - firstValue;
```

Важно: `First()` -- это самая старая запись (начало периода), `Last()` -- самая свежая (конец периода). Положительное значение означает рост курса.

### Процентное изменение

```csharp
stats.PercentChange = firstValue != 0
    ? (lastValue - firstValue) / firstValue * 100.0
    : 0;
```

Защита от деления на ноль (теоретический случай, курсы ЦБ всегда > 0).

### Текущий курс

```csharp
stats.CurrentValue = valutes.Last().Value;
```

### Количество дней

```csharp
stats.DaysCount = valutes.Count;
```

Это количество уникальных дат в выборке (может быть меньше 7 из-за выходных/праздников).

### Порог "без изменений"

Для рассылки: изменение считается нулевым, если `Math.Abs(absoluteChange) < 0.005`. Это соответствует точности форматирования `0.00` -- значение округлится до `0.00`.

## Формат дат

Для блока статистики в `/valutesevendays` используется формат `"dd MMM"` с русской локалью (`CultureInfo.GetCultureInfo("ru-RU")`), что дает "05 мар", "08 мар" и т.д.

Для заголовка рассылки: `"d MMMM"` с русской локалью, что дает "10 марта".

## Влияние на производительность

Рассылка теперь запрашивает `day=8` вместо `day=1` -- объем данных от API увеличивается в ~8 раз на каждую валюту. При 5 валютах это 5 HTTP-запросов к API, каждый возвращает ~8 записей вместо ~1. Для текущих масштабов (десятки подписчиков) влияние незначительно:

- Запрос к API: ~50-100 мс (внутренняя Docker-сеть)
- Формирование сообщения: 5 запросов x ~100 мс = ~500 мс
- Рассылка всем подписчикам: один и тот же текст, формируется один раз

Рассылка выполняется 2 раза в сутки, оптимизация не требуется.

8. **Корректность расчетов**: максимум и минимум определяются по полю `Value` с указанием дат; изменение -- разница между последним и первым элементом дедуплицированной выборки
