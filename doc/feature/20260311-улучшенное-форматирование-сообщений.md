# Улучшенное визуальное форматирование сообщений бота

## Цель

Переработать формат отображения курсов валют в Telegram-боте для повышения читаемости и информативности. Добавить эмодзи валют, человекочитаемые даты на русском, трендовые индикаторы, процентное изменение и визуальные разделители между блоками валют.

## Текущее поведение

### Формат одной валюты (команда /valuteoneday)

```
*Dollar*
 USD/RUB

 2024-03-10 ---> 85.45
```

### Формат нескольких валют за 7 дней (команда /valutesevendays)

```
*Dollar*
 USD/RUB

 2024-03-10 ---> 85.45  (- *0.25)*
 2024-03-09 ---> 85.70  (+ *0.05)*
 2024-03-08 ---> 85.65

*Euro*
 EUR/RUB

 2024-03-10 ---> 93.12  (- *0.18)*
 2024-03-09 ---> 93.30
```

### Проблемы текущего формата

1. **Нечитаемые даты**: ISO-формат `2024-03-10` сложен для быстрого восприятия
2. **Непонятное направление изменения**: `(- *0.25)*` не дает мгновенного понимания тренда
3. **Нет процентного изменения**: невозможно оценить масштаб изменения без контекста
4. **Визуально не структурировано**: блоки валют сливаются, нет четких разделителей
5. **Английские названия**: `Dollar` вместо русского `Доллар США`
6. **Ошибка в логике Difference**: разница считается "задом наперед" -- текущий день сравнивается со следующим в списке, но присваивается элементу `[i+1]`, а не `[i]`; при `day >= 3` первый элемент пропускается через `Skip(1)`, теряя самую актуальную запись

## Детали реализации

### 1. Формат даты: русские сокращения месяцев

Вместо `2024-03-10` отображать `10 мар`.

Реализация через `CultureInfo("ru-RU")`:

```csharp
// Формат: "10 мар"
valute.DateValute.ToString("dd MMM", new CultureInfo("ru-RU")).TrimEnd('.')
```

Сокращения месяцев: янв, фев, мар, апр, май, июн, июл, авг, сен, окт, ноя, дек.

### 2. Трендовые индикаторы

Стрелки показывают направление изменения курса **относительно предыдущего дня**:

| Условие | Символ | Пример |
|---------|--------|--------|
| Курс вырос (разница > 0.005) | `↑` | `↑ +0.25 (+0.29%)` |
| Курс упал (разница < -0.005) | `↓` | `↓ -0.25 (-0.29%)` |
| Без изменений (\|разница\| <= 0.005) | `→` | `→ 0.00 (0.00%)` |
| Нет данных за предыдущий день | -- | ничего не показываем |

Порог 0.005 выбран, чтобы не показывать стрелки при незначительных колебаниях из-за округления.

### 3. Абсолютное и процентное изменение

```csharp
// previousValue -- курс предыдущего дня (более ранняя дата)
// currentValue -- курс текущего дня (более поздняя дата)
double diff = currentValue - previousValue;
double percent = (diff / previousValue) * 100;

// Формат: "+0.25 (+0.29%)" или "-0.25 (-0.29%)"
string sign = diff >= 0 ? "+" : "";  // минус добавится автоматически
string formatted = $"{sign}{diff:F2} ({sign}{percent:F2}%)";
```

**Важно**: изменение считается как `текущий - предыдущий`, где "предыдущий" -- это более ранняя дата. Это исправляет текущую ошибку, где направление вычисления было инвертировано.

### 4. Визуальный разделитель

Между заголовком валюты и строками курсов используется линия из символов `━` (U+2501 BOX DRAWINGS HEAVY HORIZONTAL):

```
━━━━━━━━━━━━━━━━━
```

Длина: 17 символов. Этот символ поддерживается во всех клиентах Telegram.

Между блоками разных валют -- пустая строка (`\n`).

### 5. Заголовок блока валюты

Формат: `CHARCODE/RUB -- Name`

```
USD/RUB -- Доллар США
```

Используется `Name` из данных API (приходит на русском от ЦБ РФ). Markdown bold (`*...*`) на заголовке убирается -- заголовок и так выделяется структурой.

**Примечание**: эмодзи валют (USD, EUR, GBP, JPY, CNY и т.д.) не добавляем, чтобы сохранить чистый и единообразный вид для всех 34 поддерживаемых валют. Многие валюты не имеют подходящих эмодзи.

### 6. Выравнивание данных

Telegram использует пропорциональный шрифт, поэтому табличное выравнивание невозможно. Используем фиксированные разделители:

```
DD MMM  VALUE TREND DIFF (PERCENT%)
```

Два пробела между датой и значением, один пробел между значением и стрелкой.

#### 2.1. Исправить вычисление разницы

Текущая логика (ошибочная):
- Сравнивает `valutes[i]` с `valutes[i+1]` и записывает результат в `valutes[i+1].Difference`
- Пропускает первый элемент через `Skip(1)` при `day >= 3`
- Направление вычитания не соответствует хронологии

Новая логика:
- Данные приходят отсортированные от новых к старым (предположение -- проверить!)
- Для каждого элемента `[i]` (кроме последнего) вычислять разницу: `valutes[i].Value - valutes[i+1].Value`
- Записывать `AbsoluteDiff` и `PercentDiff` в `valutes[i]`
- Последний элемент (самая старая дата) остается без разницы (`null`)
- Убрать условие `day >= 3` для вычисления разницы -- считать всегда, когда есть минимум 2 записи

#### 2.2. Новый метод форматирования даты

```csharp
private static readonly CultureInfo RuCulture = new CultureInfo("ru-RU");

private string FormatDate(DateTime date)
{
    return date.ToString("dd MMM", RuCulture).TrimEnd('.');
}
```

Добавить `using System.Globalization;` в начало файла.

#### 2.3. Новый метод форматирования строки курса

```csharp
private string FormatValuteLine(Valute valute)
{
    var date = FormatDate(valute.DateValute);
    var value = valute.Value.ToString("F2", CultureInfo.InvariantCulture);

    if (valute.AbsoluteDiff == null)
        return $"{date}  {value}";

    var diff = valute.AbsoluteDiff.Value;
    var percent = valute.PercentDiff.Value;
    var trend = Math.Abs(diff) <= 0.005 ? "→" : (diff > 0 ? "↑" : "↓");
    var sign = diff >= 0 ? "+" : "";

    return $"{date}  {value} {trend} {sign}{diff:F2} ({sign}{percent:F2}%)";
}
```

#### 2.5. Переработать метод с массивом валют

Метод `GetValuteMessage(int day, string[] charCodesCollection, ...)`:

```csharp
public async Task<string> GetValuteMessage(int day, string[] charCodesCollection, CancellationToken cancel)
{
    var sb = new StringBuilder();

    for (int i = 0; i < charCodesCollection.Length; i++)
    {
        var valuteString = await GetValuteMessage(day, charCodesCollection[i], cancel);
        sb.Append(valuteString);

        // Пустая строка-разделитель между валютами (кроме последней)
        if (i < charCodesCollection.Length - 1)
            sb.AppendLine();
    }

    return sb.ToString();
}
```

## Риски и ограничения

1. **Пропорциональный шрифт Telegram**: точное табличное выравнивание невозможно. Принимаем как данность -- используем фиксированные пробелы.
2. **Длина сообщения**: при 5 валютах x 7 дней сообщение содержит ~40 строк. Telegram допускает до 4096 символов -- укладываемся с запасом.
3. **Сортировка данных от API**: предполагаем, что данные приходят от новых к старым. Если нет -- добавить `OrderByDescending(v => v.DateValute)` перед вычислением разницы.
4. **ParseMode.Markdown**: новый формат не использует Markdown-разметку. Если потребуется выделение (bold, italic), нужно будет переключиться на `ParseMode.Html` -- но это отдельная фича.
