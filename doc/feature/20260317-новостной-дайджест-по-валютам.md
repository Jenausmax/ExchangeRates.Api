# Новостной дайджест по валютам

## Цель

Автоматический сбор финансовых новостей из RSS-лент (ЦБ РФ, РБК, Ведомости), фильтрация по ключевым словам, дедупликация с приоритизацией "горячих" новостей, суммаризация через LLM (Polza.ai или Ollama) и рассылка подписчикам Telegram-бота.

## Ключевая идея

Дубли новостей из разных источников не выбрасываются, а повышают приоритет — чем больше источников подхватили новость, тем она "горячее" и выводится первой в дайджесте.

## Источники RSS

- `http://www.cbr.ru/rss/eventrss` — новости ЦБ РФ
- `http://www.cbr.ru/rss/RssPress` — пресс-релизы ЦБ
- `https://rssexport.rbc.ru/rbcnews/news/30/full.rss` — РБК
- `https://www.vedomosti.ru/rss/rubric/finance/markets` — Ведомости/Рынки

## LLM для суммаризации

Два варианта через абстракцию ILlmService:
1. **Polza.ai** — OpenAI-совместимый API (`https://polza.ai/api/v1`), Bearer auth, модель `deepseek/deepseek-chat`
2. **Ollama** — локальная LLM через NuGet `OllamaSharp`, модель `llama3.1:8b`

## Модели БД (SQLite, две новые таблицы)

### NewsTopic (группа одинаковых новостей)
- TopicHash (unique index) — хеш нормализованного заголовка
- HitCount — сколько источников упомянули
- Sources — "cbr,rbc,vedomosti"
- BestTitle — самый информативный заголовок
- Summary — выжимка от LLM (nullable)
- FirstSeenAt, LastSeenAt
- IsSent (bool), Priority (HitCount * 10 + RecencyBonus)

### NewsItem (каждое отдельное упоминание)
- ContentHash (index) — SHA256 нормализованного заголовка
- SourceUrl (unique) — оригинальная ссылка
- SourceName — "cbr" / "rbc" / "vedomosti"
- Title, RawContent, PublishedAt
- TopicId (FK → NewsTopic)
