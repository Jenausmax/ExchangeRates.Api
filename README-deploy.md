# Скрипт автоматизированного развертывания

## Быстрый старт

### Рекомендуемый способ (Linux/macOS)

```bash
# Скачайте скрипт
curl -fsSL https://raw.githubusercontent.com/Jenausmax/ExchangeRates.Api/main/deploy.sh -o deploy.sh

# Сделайте скрипт исполняемым
chmod +x deploy.sh

# Запустите скрипт
./deploy.sh
```

### Для Windows (Git Bash)

```bash
# Скачайте скрипт
curl -fsSL https://raw.githubusercontent.com/Jenausmax/ExchangeRates.Api/main/deploy.sh -o deploy.sh

# Запустите скрипт
bash deploy.sh
```

## Требования

- **Bash 4.0+** - встроен в Linux/macOS, доступен через Git Bash/WSL в Windows
- **Git 2.0+** - для клонирования репозитория
- **Docker 20.10+** - для контейнеризации
- **Docker Compose 1.29+** - для управления контейнерами

## Возможности скрипта

✅ **Полная автоматизация**: Клонирование, сборка, запуск одной командой
✅ **Проверка зависимостей**: Предварительная проверка git, docker, docker-compose
✅ **Интерактивная настройка**: Дружественные подсказки и валидация ввода
✅ **Валидация токена**: Проверка формата Telegram токена
✅ **Выбор режима**: Polling или Webhook режим работы бота
✅ **Автоматический .env**: Создание файла конфигурации с комментариями
✅ **Обработка ошибок**: Информативные сообщения и cleanup при ошибках
✅ **Цветной вывод**: Визуализация прогресса с иконками
✅ **Безопасность**: Скрытый ввод токена, проверка .gitignore

## Процесс развертывания

Скрипт выполняет следующие шаги:

1. **Проверка зависимостей** - проверяет git, docker, docker-compose
2. **Работа с репозиторием** - клонирует или обновляет репозиторий
3. **Настройка конфигурации** - запрашивает токен, режим, время рассылки
4. **Создание .env файла** - генерирует файл конфигурации
5. **Сборка Docker образов** - выполняет docker-compose build
6. **Запуск сервисов** - выполняет docker-compose up -d

## Управление после развертывания

После успешного развертывания используйте следующие команды:

```bash
# Просмотр логов в реальном времени
docker-compose logs -f

# Статус контейнеров
docker-compose ps

# Остановка сервисов
docker-compose down

# Пересборка и запуск
docker-compose up -d --build

# Логи только API
docker-compose logs -f exchangerates-api

# Логи только бота
docker-compose logs -f exchangerates-bot
```

## Доступ к сервисам

- **API**: http://localhost:5000
- **API (HTTPS)**: https://localhost:5001 (если настроен)

## Файл конфигурации (.env)

Скрипт создает файл `.env` в корне проекта с переменными окружения:

```bash
# Токен Telegram бота
BOT_TOKEN=YOUR_TOKEN_HERE

# Режим работы
BOT_USE_POLLING=true
BOT_WEBHOOK=

# Время рассылки
BOT_TIME_ONE=14:05
BOT_TIME_TWO=15:32
```

**ВАЖНО**: Файл `.env` содержит секреты и добавлен в `.gitignore` для безопасности.

## Troubleshooting

### Docker не найден

**Ошибка**: `Docker не установлен`

**Решение**:
- Ubuntu/Debian: `sudo apt-get install docker.io docker-compose`
- macOS: Скачайте Docker Desktop с https://docs.docker.com/docker-for-mac/install/
- Windows: Скачайте Docker Desktop с https://docs.docker.com/docker-for-windows/install/

### Нет прав доступа к Docker

**Ошибка**: `Docker daemon не запущен или нет прав доступа`

**Решение** (Linux):
```bash
sudo systemctl start docker
sudo usermod -aG docker $USER
# Выйдите и зайдите снова
```

### Некорректный формат токена

**Ошибка**: `Некорректный формат токена`

**Решение**: Получите токен у @BotFather в Telegram. Формат: `1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ`

### Порты уже заняты

**Ошибка**: `Error: bind: address already in use`

**Решение**: Остановите процесс, занимающий порты 5000/5001:
```bash
# Найти процесс
sudo lsof -i :5000
# Убить процесс
sudo kill -9 <PID>
```
